About

The Sociome Data Commons provides a repository of datasets about the social, environmental, and economic context of a patient's health. This data can help researchers and practioners identify key non-clinical factors that might affect treatment plans. The project is a collaboration between the Institue of Translational Medicine (ITM) and the Data Science Institue.

The Sociome Data Commons has assembled and integrated diverse datasets into a simple, well-documented interoperable format. Researchers can use an application programming interface (API) to access these datasets directly from code or via an interactive website. The datasets are categorized with a structured, standardized metadata model that conforms to FAIR principles.

Getting Started (Programmers)

The following instructions assume basic familiarity with Python programming. We will eventually release more accessible instructions.

sociomepy
The project utilizes a Python toolkit to aid with common data integration and harmonization steps that researchers using the data might encounter. This toolkit can access SDC datasets via API and can be run inside a clinical data enclave. We provide this toolkit as an open-source utility. Install the `sociomepy` software package by following the instructions here: https://github.com/sociome/sociomepy To summarize,
  • Sociomepy allows programmers to remote access SDC assets.
  • Sociomepy allows programmers to combine SDC assets with a variety of regional geospatial data.
  • Sociomepy allows programmers to create complex maps, charts, and models.
  • Sociomepy allows data to be exported so programmers can contribute to the SDC.


Using the SDC
Next, we overview how to find SDC assets. Click on the "Datasets" link to access the list of datasets. Users can write search queries to identify datasets of interest. When you find a dataset that you like, you can click the dataset link. That will open up a page that Follow the instructions to load data into sociomepy. Data can be downloaded as an ordinary file or accessed via API. The API access is particularly useful when you want to avoid storing a dataset permanently.
